# مسئلہ #783 تحقیقات اور فکس پلان دستاویز

## 1. مسئلے کی وضاحت (تصدیق شدہ)

- **مظہر**: جب `agents.*.model.primary/fallbacks` میں `model_name` عرف (alias) (جیسے `step-3.5-flash`) استعمال کیا جاتا ہے، تو فال بیک (fallback) سلسلہ اس عرف کو اصل `provider/model` کے طور پر پارس (parse) کرتا ہے، جس کے نتیجے میں `provider` خالی ہو سکتا ہے اور `model` غلط ہو سکتا ہے۔
- **بنیادی وجہ**: `ResolveCandidates` صرف سٹرنگز پر `ParseModelRef` انجام دیتا ہے، اور پہلے `model_list` کے ذریعے عرف کو اصل `model` فیلڈ سے میپ (map) نہیں کرتا۔
- **اثرات**:
  - فال بیک پر عمل درآمد کے دوران عرف براہ راست OpenAI سے مطابقت رکھنے والے فراہم کنندہ (provider) کو بھیجا جا سکتا ہے، جس سے `Unknown Model` کی خرابی پیدا ہوتی ہے۔
  - جب `defaults.provider` خالی ہو، تو لاگز میں `provider=` کی خالی قدر ظاہر ہوتی ہے۔

## 2. اس بار کا ہدف

- فال بیک امیدواروں کی پارسنگ کو ٹھیک کرنا: `model_list` کے ذریعے عرف کو ترجیحی طور پر پارس کرنا۔
- پرانے رویے کے ساتھ مطابقت: اگر `model_list` میں مماثلت نہ ملے، تو اصل `ParseModelRef` پر واپس جائیں۔
- اضافی ٹیسٹنگ: عرفی نام، نیسٹڈ پاتھ ماڈلز (مثلاً `openrouter/stepfun/...`)، اور خالی ڈیفالٹ فراہم کنندہ کا احاطہ کرنا۔
- کوڈ سٹائل کی تصدیق: موجودہ ریپوزٹری کے سٹائل (نام سازی، غلطیوں کی ہینڈلنگ، ٹیسٹ کا ڈھانچہ) کے ساتھ ہم آہنگی برقرار رکھنا۔

## 3. انٹرنیٹ پر بہترین طریقوں کی تحقیق کے نتائج (مکمل)

- [x] OpenAI سے مطابقت رکھنے والے گیٹ ویز (جیسے OpenRouter) کے لیے `model` فیلڈ کی تجویز کردہ ہینڈلنگ کا جائزہ لیا گیا۔
- [x] ملٹی پرووائیڈر/فال بیک ڈیزائن کے بہترین طریقوں (امیدواروں کی پارسنگ، لاگ مانیٹرنگ) کا جائزہ لیا گیا۔
- [x] بیرونی تجاویز کو اس ریپوزٹری کے قابل عمل اصولوں میں شامل کیا گیا۔

**بیرونی حوالہ جاتی نکات (OpenRouter/LiteLLM/Cloudflare AI Gateway وغیرہ کی آفیشل دستاویزات سے):**

- واضح کنفیگریشن کو ترجیح دیں، فراہم کنندہ کا اندازہ لگانے کے لیے سٹرنگ سپلٹنگ (string splitting) پر انحصار نہ کریں۔
- گیٹ وے ماڈل کی شناخت کے لیے مکمل پاتھ کی خصوصیات کو برقرار رکھیں تاکہ `Unknown Model` سے بچا جا سکے۔
- فال بیک اور پرائمری کے لیے ایک ہی پارسنگ حکمت عملی استعمال کریں تاکہ "اصل پاتھ درست اور فال بیک غلط" جیسی صورتحال سے بچا جا سکے۔

**حوالہ جاتی لنکس:**
- OpenRouter Provider Routing: https://openrouter.ai/docs/guides/routing/provider-selection
- OpenRouter Model Fallbacks: https://openrouter.ai/docs/guides/routing/model-fallbacks
- OpenRouter Chat Completion API: https://openrouter.ai/docs/api-reference/chat-completion
- LiteLLM Router Architecture: https://docs.litellm.ai/docs/router_architecture
- Cloudflare AI Gateway Chat Completion: https://developers.cloudflare.com/ai-gateway/usage/chat-completion/

**اس ریپوزٹری کے لیے قابل عمل اصول:**
- فال بیک امیدوار بنانے کے مرحلے میں پہلے `model_name -> model_list.model` میپنگ کریں۔
- اگر میپنگ نہ ملے تو پرانا پارسنگ رویہ برقرار رکھیں تاکہ مطابقت برقرار رہے۔
- "عرف + نیسٹڈ ماڈل پاتھ + خالی ڈیفالٹ پرووائیڈر" کے منظرناموں کو نئے ٹیسٹس کے ذریعے محفوظ کریں۔
