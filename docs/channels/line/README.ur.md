# Line

MalikClaw لائن (LINE) میسجنگ API اور ویب ہک (Webhook) کال بیک فنکشن کے ذریعے لائن کے لیے سپورٹ فراہم کرتا ہے۔

## کنفیگریشن

```json
{
  "channels": {
    "line": {
      "enabled": true,
      "channel_secret": "YOUR_CHANNEL_SECRET",
      "channel_access_token": "YOUR_CHANNEL_ACCESS_TOKEN",
      "webhook_path": "/webhook/line",
      "allow_from": []
    }
  }
}
```

| فیلڈ | قسم | لازمی | تفصیل |
| -------------------- | ------ | ---- | ------------------------------------------ |
| enabled | bool | ہاں | کیا لائن چینل کو فعال کرنا ہے |
| channel_secret | string | ہاں | لائن میسجنگ API کا چینل سیکریٹ (Channel Secret) |
| channel_access_token | string | ہاں | لائن میسجنگ API کا چینل ایکسیس ٹوکن (Channel Access Token) |
| webhook_path | string | نہیں | ویب ہک کا راستہ (پہلے سے طے شدہ /webhook/line ہے) |
| allow_from | array | نہیں | یوزر آئی ڈی کی وائٹ لسٹ، خالی کا مطلب ہے تمام صارفین کی اجازت ہے |

## ترتیب کا عمل

1. ایک سروس فراہم کنندہ اور میسجنگ API چینل بنانے کے لیے [LINE Developers Console](https://developers.line.biz/console/) پر جائیں
2. چینل سیکریٹ اور چینل ایکسیس ٹوکن حاصل کریں
3. ویب ہک (Webhook) ترتیب دیں:
   - لائن کے لیے ویب ہک میں HTTPS پروٹوکول کا ہونا لازمی ہے، لہذا آپ کو ایک ایسا سرور تعینات کرنا ہوگا جو HTTPS کو سپورٹ کرتا ہو، یا ngrok جیسے ریورس پراکسی ٹولز کا استعمال کریں تاکہ مقامی سرور کو پبلک نیٹ ورک پر لایا جا سکے
   - MalikClaw اب تمام چینلز کے ویب ہک کال بیکس وصول کرنے کے لیے ایک مشترکہ گیٹ وے HTTP سرور استعمال کرتا ہے، جس کا پہلے سے طے شدہ پتہ 127.0.0.1:18790 ہے
   - ویب ہک URL کو `https://your-domain.com/webhook/line` پر سیٹ کریں، اور پھر بیرونی ڈومین کو لوکل گیٹ وے (ڈیفالٹ پورٹ 18790) پر ریورس پراکسی کریں
   - ویب ہک کو فعال کریں اور URL کی تصدیق کریں
4. چینل سیکریٹ اور چینل ایکسیس ٹوکن کو کنفیگریشن فائل میں درج کریں
