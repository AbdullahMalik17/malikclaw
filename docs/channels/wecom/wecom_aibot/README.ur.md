# کاروباری وی چیٹ (WeCom) اسمارٹ روبوٹ (AI Bot)

کاروباری وی چیٹ اسمارٹ روبوٹ (AI Bot) کاروباری وی چیٹ کا سرکاری فراہم کردہ AI مکالمہ تک رسائی کا طریقہ ہے، جو نجی اور گروپ چیٹس دونوں کو سپورٹ کرتا ہے، اس میں اسٹریمنگ رسپانس پروٹوکول (streaming response protocol) شامل ہے، اور ٹائم آؤٹ کے بعد `response_url` کے ذریعے حتمی جواب کو فعال طور پر پش کرنے کی سہولت بھی موجود ہے۔

## دیگر WeCom چینلز کے ساتھ موازنہ

| خصوصیت | WeCom Bot | WeCom App | **WeCom AI Bot** |
|------|-----------|-----------|-----------------|
| نجی چیٹ | ✅ | ✅ | ✅ |
| گروپ چیٹ | ✅ | ❌ | ✅ |
| اسٹریمنگ آؤٹ پٹ | ❌ | ❌ | ✅ |
| ٹائم آؤٹ کے بعد پش | ❌ | ✅ | ✅ |
| کنفیگریشن کی پیچیدگی | کم | زیادہ | درمیانی |

## کنفیگریشن

```json
{
  "channels": {
    "wecom_aibot": {
      "enabled": true,
      "token": "YOUR_TOKEN",
      "encoding_aes_key": "YOUR_43_CHAR_ENCODING_AES_KEY",
      "webhook_path": "/webhook/wecom-aibot",
      "allow_from": [],
      "welcome_message": "ہیلو! میں آپ کی کیا مدد کر سکتا ہوں؟",
      "max_steps": 10
    }
  }
}
```

| فیلڈ             | قسم   | لازمی | تفصیل                                               |
| ---------------- | ------ | ---- | -------------------------------------------------- |
| token            | string | ہاں   | کال بیک کی تصدیق کا ٹوکن، جو AI Bot مینجمنٹ پیج پر سیٹ کیا جاتا ہے |
| encoding_aes_key | string | ہاں   | 43 حروف پر مشتمل AES کلید، جو AI Bot مینجمنٹ پیج پر خودکار طور پر تیار کی جاتی ہے |
| webhook_path     | string | نہیں   | Webhook کا راستہ (ڈیفالٹ: /webhook/wecom-aibot)         |
| allow_from       | array  | نہیں   | صارف ID کی سفید فہرست، خالی ارے کا مطلب ہے تمام صارفین کی اجازت ہے |
| welcome_message  | string | نہیں   | جب صارف چیٹ شروع کرے تو بھیجا جانے والا خوش آمدیدی پیغام، خالی چھوڑنے پر کوئی پیغام نہیں جائے گا |
| reply_timeout    | int    | نہیں   | جواب کا ٹائم آؤٹ (سیکنڈز میں، ڈیفالٹ: 5)                        |
| max_steps        | int    | نہیں   | ایجنٹ کے زیادہ سے زیادہ اقدامات کی تعداد (ڈیفالٹ: 10)                   |

## سیٹ اپ کا طریقہ کار

1. [کاروباری وی چیٹ مینجمنٹ کنسول](https://work.weixin.qq.com/wework_admin) میں لاگ ان کریں۔
2. "ایپ مینجمنٹ" (Application Management) → "اسمارٹ روبوٹ" (Smart Robot) پر جائیں، اور ایک نیا AI Bot بنائیں یا کسی موجودہ کو منتخب کریں۔
3. AI Bot کے کنفیگریشن پیج پر، "پیغامات وصول کریں" (Receive Messages) کی معلومات درج کریں:
   - **URL**: `http://<your-server-ip>:18791/webhook/wecom-aibot`
   - **Token**: خودکار طور پر تیار کریں یا اپنی مرضی کا درج کریں۔
   - **EncodingAESKey**: "خودکار طور پر تیار کریں" پر کلک کریں، آپ کو 43 حروف کی کلید مل جائے گی۔
4. Token اور EncodingAESKey کو MalikClaw کی کنفیگریشن فائل میں درج کریں، سروس شروع کرنے کے بعد مینجمنٹ کنسول میں واپس جا کر سیو کریں (کاروباری وی چیٹ ایک تصدیقی درخواست بھیجے گا)۔

> [!TIP]
> آپ کے سرور تک کاروباری وی چیٹ کے سرور کی رسائی ہونا ضروری ہے۔ اگر آپ لوکل ڈویلپمنٹ کر رہے ہیں، تو [ngrok](https://ngrok.com) یا frp کا استعمال کر سکتے ہیں۔

## اسٹریمنگ رسپانس پروٹوکول

WeCom AI Bot "اسٹریمنگ پل" (streaming pull) پروٹوکول استعمال کرتا ہے، جو عام ویب ہک کے ایک بار کے جواب سے مختلف ہے:

```
صارف کا پیغام
  │
  ▼
MalikClaw فوری طور پر {finish: false} واپس کرتا ہے (ایجنٹ کام شروع کر دیتا ہے)
  │
  ▼
کاروباری وی چیٹ ہر 1 سیکنڈ کے بعد {msgtype: "stream", stream: {id: "..."}} پل کرتا ہے
  │
  ├─ ایجنٹ کا کام جاری ہے → {finish: false} واپس کریں (انتظار جاری رکھیں)
  │
  └─ ایجنٹ کا کام مکمل ہو گیا → {finish: true, content: "جواب کا متن"} واپس کریں
```

**ٹائم آؤٹ ہینڈلنگ** (30 سیکنڈ سے زیادہ وقت لگنے پر):

اگر ایجنٹ کو جواب دینے میں تقریباً 30 سیکنڈ سے زیادہ وقت لگے (کاروباری وی چیٹ کی زیادہ سے زیادہ پولنگ ونڈو 6 منٹ ہے)، تو MalikClaw یہ کرے گا:

1. فوری طور پر اسٹریم بند کر دے گا اور صارف کو یہ پیغام دکھائے گا: "⏳ پروسیسنگ جاری ہے، براہ کرم انتظار کریں، نتیجہ تھوڑی دیر میں بھیج دیا جائے گا۔"
2. ایجنٹ پس منظر (background) میں کام جاری رکھے گا۔
3. ایجنٹ کا کام مکمل ہونے کے بعد، پیغام میں موجود `response_url` کے ذریعے حتمی جواب صارف کو فعال طور پر پش کر دیا جائے گا۔

> `response_url` کاروباری وی چیٹ کی طرف سے جاری کیا جاتا ہے، یہ 1 گھنٹے کے لیے کارآمد ہوتا ہے اور اسے صرف ایک بار استعمال کیا جا سکتا ہے۔ اسے انکرپٹ کرنے کی ضرورت نہیں ہوتی، بس براہ راست POST markdown باڈی بھیجیں۔

## خوش آمدیدی پیغام

اگر `welcome_message` کنفیگر ہو، تو جب بھی صارف AI Bot کے ساتھ چیٹ ونڈو کھولے گا (`enter_chat` ایونٹ)، MalikClaw خود بخود یہ پیغام بھیجے گا۔ خالی چھوڑنے کی صورت میں اسے نظر انداز کر دیا جائے گا۔

```json
"welcome_message": "ہیلو! میں MalikClaw AI اسسٹنٹ ہوں، میں آپ کی کیا مدد کر سکتا ہوں؟"
```

## عام مسائل

### کال بیک URL کی تصدیق ناکام

- تصدیق کریں کہ سرور کا فائر وال متعلقہ پورٹ (ڈیفالٹ 18791) کے لیے کھلا ہے۔
- تصدیق کریں کہ `token` اور `encoding_aes_key` درست درج کیے گئے ہیں۔
- چیک کریں کہ MalikClaw کے لاگز میں کاروباری وی چیٹ سے GET درخواست موصول ہوئی ہے۔

### پیغام کا کوئی جواب نہیں ملا

- چیک کریں کہ کہیں `allow_from` نے بھیجنے والے پر پابندی تو نہیں لگائی۔
- لاگز میں دیکھیں کہ کہیں `context canceled` یا ایجنٹ کی کوئی غلطی تو موجود نہیں ہے۔
- تصدیق کریں کہ ایجنٹ کی کنفیگریشن (`model_name` وغیرہ) درست ہے۔

### طویل کاموں کے بعد حتمی جواب موصول نہیں ہوا

- تصدیق کریں کہ کال بیک میں `response_url` موجود تھا (یہ صرف کاروباری وی چیٹ کے نئے AI Bot ورژن میں دستیاب ہے)۔
- تصدیق کریں کہ آپ کا سرور انٹرنیٹ تک رسائی رکھتا ہے (تاکہ `response_url` پر POST درخواست بھیج سکے)۔
- لاگز میں `response_url mode` اور `Sending reply via response_url` جیسے الفاظ تلاش کریں۔
